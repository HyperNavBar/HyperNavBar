# 使用教程

本文档介绍 HyperNavBar 的基本使用方法，帮助快速上手。

<br>

## 首次使用

首次打开 HyperNavBar 时，规则列表为空。按以下步骤开启预设规则：

1. 点击右上角 **预设规则** 图标（位于右上角 `+`左边）
2. 选择你需要的规则源（官方配置源或社区配置源），点击 **添加**
3. 回到主页，点击 **立即应用** 即可生效

<br>

## 新增规则

除预设规则外，还可以通过以下方式新增规则：

- **添加云端订阅** — 点击右上角 `+`，输入云端 JSON 订阅地址
- **添加本地规则** — 点击右上角 `+`，选择 **本地规则**，在编辑器中编写 JSON 或通过可视化编辑器创建

新增的规则会出现在规则列表中，可通过拖拽调整优先级顺序，点击规则卡片即可管理其内容。

<br>

## 规则生效逻辑

规则列表的顺序决定优先级：**上方规则优先级高于下方**。

当多条规则包含同一个应用和活动的配置时，以优先级最高的那条为准。

**注意事项：**

- **官方配置源** 用于恢复使用 HyperNavBar 之前的系统默认状态，或作为自定义修改的基础模板。**请勿**将官方配置源与社区配置源同时使用，二者只需保留其一。
- **社区配置源** 由 [HyperNavBarRules](https://github.com/HyperNavBar/HyperNavBarRules) 仓库维护，覆盖了常用应用的适配，一般情况下直接使用社区配置源即可。

**推荐配置方式：**

将社区配置源（或官方配置源）的优先级放到最低（列表底部），通过在上方新增本地规则来实现个性化适配。这样既能保留完整的基础配置，又能灵活自定义。

<br>

## 本地规则：悬浮窗快捷添加

使用悬浮窗识别功能可以快速添加本地规则：

1. 在可视化编辑器中点击 **悬浮窗识别** 按钮
2. 切换到需要适配的应用，悬浮窗会实时显示该应用的包名、应用名称和活动 ID
3. 点击 **一键添加**，该应用信息会自动填入编辑器并打开编辑面板
4. 选择适配模式和参数后保存即可

悬浮窗中的信息可单独点击复制，历史识别记录可通过 **历史记录** 按钮查看。

<br>

## 屏幕取色

在可视化编辑器的活动规则编辑界面，自定义颜色模式可以点击按钮打开屏幕取色器：

1. 点击 **屏幕取色** 按钮，打开截屏悬浮窗
2. 切换到目标应用界面，点击截图
3. 在屏幕上移动十字准星到取色位置，点击确认，拾取到的颜色值会自动填入颜色输入框

该功能适用于需要精确指定导航栏颜色的场景。

<br>

## 常用配置方法

以下通过可视化编辑器说明几种常见场景的配置方式。

### 保持应用原始行为（不修改）

如需对某个应用禁用沉浸适配，保持其原有导航栏样式：

1. 在可视化编辑器中添加该应用，进入活动规则编辑
2. 将 **沉浸模式** 设为 **禁用模式**（mode = 0）

适用于应用自身已适配良好、存在兼容性问题或全屏视频播放等场景。

### 悬浮模式配置

悬浮模式会隐藏导航栏背景，实现完全沉浸，但导航栏区域会遮挡下方内容：

1. 将 **沉浸模式** 设为 **悬浮模式**（mode = 2）
2. 对话框模式和弹窗模式按需设置，一般保持默认即可

适用于信息流页面（微博、知乎等）、搜索页、设置页等无底部固定控件的场景。

### 取色配置（视图采样与 SF 采样）

取色模式（mode = 1）下有两种采样方式，颜色来源优先级为：

**自定义颜色 > SF 采样 > 视图采样 > 系统默认**

即：如果同时配置了 SF 采样和视图采样，SF 采样优先生效。

#### 视图采样

系统分析应用视图树的颜色，精度高但性能开销稍大。适用于大多数场景：

1. 将 **沉浸模式** 设为 **取色模式**（mode = 1）
2. **导航栏颜色** 保持 **自动取色**（color = 1）
3. **SF 采样模式** 保持 **自动模式**（sf_sampling_mode = 0）

#### SF 采样

直接从屏幕像素取色，性能好但精度略低，且略有延迟。适用于底部颜色接近纯色、变化频率低的场景（如不同皮肤切换颜色）：

1. 将 **沉浸模式** 设为 **取色模式**（mode = 1）
2. 将 **SF 采样模式** 设为 **强制启用**（sf_sampling_mode = 1）

由于 SF 采样优先级高于视图采样，对于不需要 SF 采样的场景，**务必**将 SF 采样模式设为 **自动模式**（sf_sampling_mode = 0），以确保视图采样正常工作。

### 对话框与弹窗的独立配置

对话框和弹窗的沉浸行为可单独控制，与主界面配置互不影响：

- **对话框模式** — 控制对话框窗口的沉浸适配，可选禁用、视图采样或 SF 采样
- **弹窗模式** — 控制弹窗窗口的沉浸适配，选项与对话框模式相同

例如，主界面使用 SF 取色，但对话框使用视图采样，可分别设置。

### 自定义固定颜色

如需将导航栏设为指定颜色：

1. 将 **沉浸模式** 设为 **取色模式**（mode = 1）
2. 在 **导航栏颜色** 中选择 **自定义颜色**，输入通过取色、输入色值等方式设置颜色

## 参与贡献

欢迎为 [HyperNavBarRules](https://github.com/HyperNavBar/HyperNavBarRules) 贡献适配规则。

**推荐工作流程：**

1. 在 HyperNavBar 中新建一个本地规则，通过悬浮窗识别和可视化编辑完成适配
2. 将该本地规则的 JSON 导出
3. 在 HyperNavBarRules 仓库提交 Pull Request，将你的规则 JSON 放入 `merge.json` 中
4. 发版时会自动合并，你的更改将出现在社区配置源中，所有用户均可受益

提交 Issue 或 PR 前，请在本地设备上完成测试，确保适配效果正常。
